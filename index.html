🪙 CoinFlip Verify

Every flip is provably fair. Before the round, a commit hash (SHA-256 of the secret seed) is published. After the flip, the seed is revealed. Use this tool to verify the result was predetermined — HEADS or TAILS, 50/50, no manipulation possible.

How It Works

Before each round, the server generates a random 32-byte seed and publishes its SHA-256 hash (the commit hash). Players place bets only seeing the hash. After the round, the seed is revealed — anyone can verify it matches the hash and recompute the result.

The result is HEADS if the first byte of the result hash is less than 128, and TAILS otherwise. This gives an exact 50/50 probability (128 out of 256 possible byte values).

// 1. Commit (before round)
commit_hash = SHA256(server_seed)

// 2. Flip result (deterministic)
input = server_seed + "flip" + round_number_le64
hash = SHA256(input)
v = hash[0] // first byte (0-255)
result = v < 128 ? HEADS : TAILS

// Probability: 128/256 = exactly 50%
